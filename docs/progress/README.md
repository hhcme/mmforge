# Progress Reports

This directory stores one handoff report per completed goal-mode task.

When an implementation agent finishes a goal, it must create a new Markdown report here instead of only replying in chat.

## Naming

Use:

```text
YYYY-MM-DD-target-name.md
```

Examples:

```text
2026-06-29-repository-foundation.md
2026-06-29-lsm-runtime-model.md
```

## Required Template

```text
# <Goal Name>

Date:
Agent:
Target:

## Summary

## Modified Files

## Architecture Decisions

## Key Algorithms

## Commands Run

## Checks Not Run

## Known Issues

## Next Target Recommendation

## Review Focus For Codex

## Screenshots / Logs

## Sample Files / testfile Usage

## New Dependencies And Licenses
```

Do not overwrite old reports. If a goal needs another pass, create a second report with a suffix such as `-round-2`.
